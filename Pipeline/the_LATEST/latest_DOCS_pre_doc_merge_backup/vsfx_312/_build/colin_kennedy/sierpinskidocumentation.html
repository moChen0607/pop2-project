Wrapper¶

To facilitate wrapping the generated objects to a deforming mesh, I created another tool to very quick-and-dirty wrap and connect all of the objects together under a single interface (the wrapperCtl)

sierpinski.sierpinski.get_tetrahedron(node='pPyramid1')¶

Quick casts an existing tetrahedron to a PyNode and then retrurns its shape. Assumes its name is ‘pPyramid1’ by default if nothing is specified

Parameters:node (str) – The name of the node to cast to a PyNode object
Returns:The shape of the casted PyNode object
Return type:(<pymel.core.nodetypes.Mesh>)
sierpinski.sierpinski.init_tetrahedron()¶

Quick function to create a tetrahedron (poly pyramid)

Returns:A mesh pointing to the tetrahedron
Return type:(<pymel.core.nodetypes.Mesh>)
sierpinski.sierpinski.make_and_set_objs(points, scl, grp=None)¶

A convenience function to create and set objects from the sierpinski data. Right now the objects that it creates are locators but it can be anything. The resulting objects are then grouped if specified.

Parameters:
  • points (iterable) – Represents all of the objects to create. If points contains meaningful data, it can be used to set the object’s translate/rotate/scale and other attributes
  • scl (list) – Requires a list with length of 3 to set the scale
  • grp (<pymel.code.nodetypes.Transform>) – A pointer to the group to parent all of the newly created objects
Returns:

None
Return type:

NoneType
sierpinski.sierpinski.midpoint_3d(p1, p2)¶

Gets the midpoint of two iterables in an efficient one liner. The function is just for syntax sugar

Parameters:
  • p1 (list) – A list or iterable of points to get the midpoint from
  • p2 (list) – A list or iterable of points to get the midpoint from
Returns:

The output midpoint between the two points
Return type:

list or iterable
sierpinski.sierpinski.sierpinski(seed, verts, reps)¶

Creates sierpinski pattern from a at least 3 points

Parameters:
  • seed (list) – Any 3D point. [0, 0, 0] is fine in most cases
  • verts (list of lists) – A list containing 3D position points in world space. Represents every point in a base tetrahedron
  • reps – (int): The number of points to generate from the sierpinski
Returns:

The created points from the sierpinski algorithm
Return type:

generator
sierpinski.wrapper.wrap_objects(wrapper, wrapperCtl, selection)¶

Wrap objects to a single wrapper object and creates a couple interface controls all of the wrapped objects (so that the wrap can be toggled on/off more easily)

Note

I had to dig quite a bit to find CreateWrap, which actually isn’t a pymel command, which is why the call uses cmds. Also, the command returns None, even on success, so I had to manually refind the wrap node that was created by searching the selection object’s history. Thankfully history is listed most-recent to oldest so I’m able to get away with that without accidentally picking up a wrap node that I didn’t intend to

Parameters:
  • wrapper (<pymel.core.nodetypes.Mesh>) – The shape to wrap the selection to
  • wrapperCtl (<pymel.core.nodetypes.Transform>) – Convenience controller
  • selection (<pymel.core.nodetypes.Transform>) – A iterable of object transforms
Returns:

None
Return type:

NoneType
